Day 19 / Function Workflow

函数真正开始有价值的时刻，是多个函数不再各自孤立，而是共同组成一个清晰流程。

今天你会把输入、判断、输出拆成不同函数，让程序从“堆在一起的一大段代码”逐渐变成有结构的流程。

函数配合的基础结构

每个函数只承担一段职责。

def get_score(): return int(input("请输入分数：")) def judge_score(score): return "及格" if score >= 60 else "不及格" def show_result(result): print(result)

1. 今天目标

今天的目标是学会把一个小程序拆成多个职责清晰的函数，明白“结构化”比单纯能跑更重要。

2. 核心概念

如果输入、判断、输出全部写在一起，代码短时还能忍，稍微一长就会难改。函数综合训练的核心，就是拆责任。

Input

输入函数负责拿数据

例如从用户那里获取分数、姓名或命令，不顺手做太多别的事。

Logic

处理函数负责判断和计算

例如判断成绩是否及格、统计数量、整理字符串。

3. 示例代码

下面这个例子把一个成绩判断流程拆成 3 段。

def get_score(): return int(input("请输入分数：")) def judge_score(score): if score >= 60: return "及格" return "不及格" def show_result(result): print("结果：", result) score = get_score() result = judge_score(score) show_result(result)
示例在说明什么： 程序仍然只是做一次成绩判断，但结构已经比“全写在一坨里”清楚很多。

4. 关键提醒

  • 一个函数尽量只做一件事。
  • 函数名最好能直接说明用途。
  • 主流程里只保留“先做什么，再做什么”的骨架。
  • 这会让后面改逻辑时更容易定位。
今天最该抓住的点： 当你觉得一段程序越来越长时，就应该开始问自己：哪些部分可以拆成单独函数？

5. 今日练习

P1

拆分成绩程序

把前面的成绩判断逻辑拆成输入、判断、输出 3 个函数。

P2

拆分欢迎程序

把“读取名字”“拼接欢迎语”“输出结果”拆成不同函数。

P3

给函数改好名字

把你以前写过的一个练习拿出来，检查函数名是否真的一眼能看懂。

6. 今日总结

  • 多个函数配合是程序结构化的开始。
  • 输入、处理、输出尽量分开。
  • 函数越职责清晰，程序越容易维护。
  • 结构化思维会直接影响后面的项目质量。
今天过关标准： 你能把一个 8 到 15 行的小程序拆成 2 到 4 个清晰函数，并解释每个函数负责什么，就说明今天学到了真正有价值的部分。

1. 今天目标

今天的目标是做出一个最小成绩管理器：至少能保存学生姓名和分数，并显示结果或统计信息。

2. 核心概念

项目阶段不要被“功能越多越好”误导。真正关键的是：数据结构明确、函数划分清楚、流程能从头到尾跑通。

Data

每条成绩记录是一条资料

最自然的写法通常是列表里放多个字典，每个字典保存 name 和 score。

Action

先做核心功能

例如添加记录、显示全部、统计及格人数，不要一开始就上很多菜单。

3. 示例代码

下面这个骨架已经足够成为一个小项目的起点。

students = [] def add_student(students, name, score): students.append({"name": name, "score": score}) def show_students(students): for student in students: print(student["name"], student["score"])
示例在说明什么： 有了清楚的数据结构和两个核心函数，后面再加统计、分类、保存文件就会容易很多。

4. 关键提醒

  • 先把“添加”和“显示”做稳，再加统计。
  • 数据结构一开始就设计清楚，后面会省很多事。
  • 函数尽量职责单一。
  • 如果要扩展，可以再接文件保存。
今天最该抓住的点： 项目阶段最重要的是“做出来并能继续改”，不是一上来就追求复杂菜单系统。

5. 今日练习

P1

最小成绩管理器

至少实现添加学生记录和显示全部记录两个功能。

P2

统计及格人数

在显示基础上，加一个函数统计分数大于等于 60 的人数。

P3

扩展平均分

再补一个平均分函数，感受项目如何一步步扩展。

6. 今日总结

  • 项目训练重点是功能闭环和结构清晰。
  • 列表 + 字典是很自然的成绩记录结构。
  • 先做最核心功能，再逐步扩展。
  • 这个项目会直接体现你前面基础是否打实。
今天过关标准： 你能完成一个最小成绩管理器，并让它至少支持添加和显示功能，就说明你已经开始真正进入项目思维。

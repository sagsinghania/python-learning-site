1. 今天目标

今天的目标是知道异常不是“奇怪报错”，而是程序运行时出现的问题，并学会用 try / except 处理最常见的输入错误。

2. 核心概念

前面的大多数示例都默认输入正确、文件存在、数据格式正常。现实里不是这样。异常处理就是程序的保护层。

Try

先尝试执行

把可能出问题的代码放进 try，让程序先试着运行。

Except

出错时走备用方案

如果 try 里的代码真的出错，就转到 except，给出提示或采取补救动作。

3. 示例代码

这个例子展示了：当用户输入不是数字时，程序不会直接崩掉，而是给出明确提示。

try: score = int(input("请输入分数：")) print("你输入的是：", score) except ValueError: print("请输入整数分数，不要输入文字")
示例在说明什么： 异常处理的重点，是把不可控输入变成可控反馈。

4. 关键提醒

  • 最常见的入门异常之一是 ValueError。
  • try 里放“可能出错”的代码。
  • except 里写“出错后怎么处理”。
  • 异常处理不是掩盖问题，而是让程序以更友好的方式回应问题。
今天最该抓住的点： 你不用一口气记住很多异常类型，零基础阶段先把输入转换和文件操作相关的异常处理好就够了。

5. 今日练习

P1

年龄输入保护

让程序接收年龄输入，并用 try / except 处理用户输入文字的情况。

P2

分数输入保护

把成绩判断程序改成带异常处理的版本。

P3

尝试读不存在的文件

查一下读取不存在文件时会发生什么，再思考该怎样给用户更友好的提示。

6. 今日总结

  • 异常是程序运行时出现的问题。
  • try 用来包住可能出错的代码。
  • except 用来写出错后的处理方式。
  • 异常处理能让程序更稳、更友好。
今天过关标准： 你能自己写出一个带 try / except 的输入保护程序，并解释为什么它比直接 int(input()) 更稳，就说明今天掌握到位。

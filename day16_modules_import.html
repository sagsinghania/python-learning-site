1. 今天目标

今天的目标是知道“模块 = 可复用的一块代码”，并学会用 import 把现成能力接到自己的程序里。

2. 核心概念

前面你写的大多还是单文件练习。今天开始，程序会更接近真实开发方式：把功能拆开，按需导入。

Lib

模块就是一块可复用代码

它可以是 Python 自带的标准库，也可以是你自己写的另一个文件。

Use

import 就是把能力引进来

你不需要重写平方根、随机数之类功能，可以直接导入已有模块。

3. 示例代码

下面这个例子说明：导入后，要通过模块名去访问其中的功能。

import random number = random.randint(1, 10) print(number)
示例在说明什么： 导入 random 后，就能调用它里面的 randint() 来生成随机整数。

4. 关键提醒

  • import 后通常要通过“模块名.功能名”访问。
  • 标准库是 Python 自带的一组常用模块。
  • 模块名写错或文件不存在时，程序会报导入错误。
  • 今天先把“会导入并使用”练稳，不急着学复杂包结构。
今天最该抓住的点： 模块和 import 的核心价值，是减少重复造轮子，并为后面多文件组织打基础。

5. 今日练习

P2

导入 random

随机输出一个 1 到 100 的整数，感受“借用外部能力”的效果。

P3

想象自定义模块

写下你觉得以后适合拆成单独文件的功能，比如欢迎语、成绩判断或工具函数。

6. 今日总结

  • 模块是一块可复用代码。
  • import 可以把标准库或自定义模块的能力拿来用。
  • 导入后通常通过“模块名.功能名”访问。
  • 这是程序开始组织结构的重要一步。
今天过关标准： 你能自己导入至少 2 个标准库模块，并解释为什么导入比自己重写更合理，就说明今天的目标达成了。

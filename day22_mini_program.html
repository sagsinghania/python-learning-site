1. 今天目标

今天的目标是完成一个小型综合程序，让你开始具备“组织一个完整练习”的能力，而不是只会写散碎片段。

2. 核心概念

综合程序不一定要很大，但至少应该有数据结构、函数组织、输入输出和基本判断。这样才算真正接近小工具。

Data

先想清楚数据怎么放

例如待办事项适合列表 + 字典，分数记录也可以用类似结构。

Flow

再安排主流程

例如添加数据、显示数据、处理状态、必要时保存结果。

3. 示例代码

下面这个极简待办例子，已经体现了“数据结构 + 函数组合”的雏形。

tasks = [] def add_task(tasks, task): tasks.append({"task": task, "done": False}) def show_tasks(tasks): for item in tasks: print(item["task"], item["done"]) add_task(tasks, "完成 Python 学习") show_tasks(tasks)
示例在说明什么： 虽然程序还很小，但已经有了数据设计、功能拆分和主流程调用，这就是综合程序思路。

4. 关键提醒

  • 综合程序先求结构清晰，不求功能繁多。
  • 先把主流程跑通，再考虑加细节。
  • 数据结构选对，后面很多逻辑会自然顺很多。
  • 函数名、变量名和输出信息都要尽量清楚。
今天最该抓住的点： 做综合练习时，最常见的问题不是语法不会，而是没有先设计数据和流程。

5. 今日练习

P1

最小待办工具

创建一个任务列表，至少支持添加任务和显示任务。

P2

最小成绩记录

创建一个列表保存多个分数，再写函数显示所有分数和及格人数。

P3

加入异常保护

给输入环节加一层 try / except，让程序更稳。

6. 今日总结

  • 综合程序强调的是结构组合能力。
  • 先设计数据，再设计流程。
  • 函数、列表、字典和判断会在综合练习里频繁一起出现。
  • 这一步是进入更完整项目前的重要过渡。
今天过关标准： 你能做出一个至少包含“数据结构 + 函数 + 循环/判断”的小练习，并确保它能从头跑通，就说明今天真正掌握住了第三阶段的主线。

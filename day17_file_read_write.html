1. 今天目标

今天的目标是学会文本文件最基础的打开、写入和读取，明白为什么文件让程序拥有了“记忆”。

2. 核心概念

如果程序一关闭，所有信息都消失，那很多工具都没有价值。文件读写正是程序把数据保存下来、下次继续使用的基础。

Read

读取文件内容

把已经保存在文件里的文字拿回来，交给程序继续处理。

Write

把内容写入文件

让程序运行后把结果保存到磁盘，而不是只在屏幕上显示一次。

3. 示例代码

下面这个例子把一段学习记录写入文件，然后再读回来。

with open("study_log.txt", "w", encoding="utf-8") as file: file.write("Day 17: 文件读写\n") with open("study_log.txt", "r", encoding="utf-8") as file: content = file.read() print(content)
示例在说明什么： 程序先把文字写进文件，再读取同一个文件，因此数据不只是停留在终端输出里。

4. 关键提醒

  • w 模式通常表示写入，会覆盖原内容。
  • r 模式表示读取。
  • with 的好处是文件用完后会自动关闭。
  • 编码写成 utf-8 可以更稳地处理中英文文本。
今天最该抓住的点： 文件读写的核心不是背模式字母，而是理解：程序开始和外部持久数据打交道了。

5. 今日练习

P1

写一份学习日志

把今天的学习主题写入一个 txt 文件，然后打开文件确认内容真的保存成功。

P2

读取文件内容

读取你刚写入的文件，并把内容 print() 出来。

P3

改写成两行内容

让程序连续写入两行文字，感受文件不是只能存一个值。

6. 今日总结

  • 文件让数据在程序结束后依然保留。
  • 读写文本时常见模式是 r 和 w。
  • with 是打开文件的推荐写法。
  • 文件是后面小项目非常重要的数据基础。
今天过关标准： 你能自己写出一个“写文件 + 读文件 + 输出结果”的最小程序，就说明今天的内容已经真正掌握。

当程序开始和文件、输入打交道时，报错也会更常出现。

第 18 天会进入异常处理。你会学会用 try / except 保护程序，让它在输入错误或文件问题出现时不至于直接崩掉。

1. 核心命令

先掌握这些高频命令，你看很多入门代码时就不会再是完全陌生的状态。这里不要求你全背下来，只要求先看熟、先知道作用。

print() 输出

把内容显示到屏幕上

它的作用就是告诉程序：“把这句话、这个数字或者这个结果打印出来给我看。”

print("你好，Python")
input() 输入

从用户那里拿到信息

它会先停下来，等你输入内容，再把输入的东西交给程序继续处理。

name = input("请输入名字：")
type() 类型

查看数据属于什么类型

当你不确定一个值是字符串还是数字时，可以用它来检查。

print(type(18))
if 如果

满足条件时执行某件事

它像在问程序：“如果这个条件成立，那你就这样做。”

if score >= 60: print("及格")
else 否则

条件不成立时怎么做

当 `if` 里的条件不满足，就执行 `else` 里面的内容。

else: print("不及格")
for 循环

重复做同一类事情

当你不想把同样的代码写很多次时，就会用到循环。

for i in range(3): print("继续练习")
先抓住动作，不要先背语法： 看到 `print` 想到“输出”，看到 `input` 想到“接收输入”，看到 `if` 想到“条件判断”，这一步就够了。

2. 常用英文

这些英文会在入门阶段高频出现。你不用一次性全部背会，先做到“看见它就知道大概是什么意思”。

W1

数据相关

variable = 变量

可以把它理解成一个有名字的盒子。

value = 值

盒子里装着的内容就是值。

string = 字符串

被引号包起来的一段文字。

number = 数字

包括整数和小数。

W2

判断相关

condition = 条件

程序做判断时要先看条件。

True = 真

表示条件成立。

False = 假

表示条件不成立。

compare = 比较

比如大于、小于、是否相等。

W3

动作相关

output = 输出

让程序把结果显示出来。

input = 输入

从用户那里接收内容。

loop = 循环

重复做某件事。

function = 函数

把一段逻辑打包，方便重复使用。

Tip

零基础怎么记这些词更有效

不要把它们当成纯英语单词去硬背，而是把它们和“程序动作”绑在一起记。

  • `print` 一看到就想到“输出”。
  • `input` 一看到就想到“等用户输入”。
  • `if` 一看到就想到“如果满足条件”。
  • `for` 一看到就想到“重复做几次”。

4. 学习总览

对零基础用户来说，最容易卡住的不是“不会敲代码”，而是不知道从哪里开始、为什么先学这个、学到哪里算入门。现在这些问题都收在这一页里，你只需要顺着往下走。

01

Python 是什么

Python 是一门可读性强、入门门槛相对低、应用场景很广的编程语言。它既适合零基础学习，也能一路延伸到自动化、数据分析、网站开发和 AI。

02

你现在最该学什么

不是一下子去碰复杂项目，而是先学会高频命令、变量、输入输出、运算符和判断逻辑。前面这些会了，后面大部分内容都会更顺。

03

如何判断自己在进步

你不需要一开始就写长程序。只要能看懂简单代码、能自己写出 5 到 10 行的小程序、能修改别人给的例子，就已经是在有效进步。

推荐节奏： 每天 30 到 50 分钟，按“看概念 → 看例子 → 自己改代码 → 做 1 到 3 题练习”的顺序推进，不要只看不动手。

5. 30 天学习路线

这份路线不是为了让你机械打卡 30 次，而是把入门阶段最核心的内容拆成四个阶段，让你知道每一段到底在解决什么问题。

阶段 1：第 1 到 7 天，建立编程感觉

重点是看懂命令、理解变量和输入输出，第一次写出能运行的小程序。

Day 1
  • 认识程序与顺序执行
  • 学会 `print()`
  • 理解代码就是明确步骤
Day 2
  • 理解变量的作用
  • 保存名字、价格、分数
  • 会修改变量再输出
Day 3
  • 认识 `str`、`int`、`float`
  • 会用 `type()` 看类型
  • 理解字符串和数字区别
Day 4
  • 学会 `input()` 和 `print()` 配合
  • 做第一个交互程序
  • 开始体验人机互动

阶段 2：第 8 到 15 天，强化基础语法

重点是循环、列表、字典、字符串处理，让程序从“一次性动作”升级成能处理更多数据。

for / while list / dict 字符串处理 函数入门

阶段 3：第 16 到 23 天，开始组织代码

重点是函数、模块、文件操作和异常处理。你会逐渐明白，程序不只是“写一段能跑的代码”，还要学会整理和复用。

def import open() try / except

阶段 4：第 24 到 30 天，进入小项目

重点是把前面的知识拼起来，做出完整的小项目。哪怕项目简单，只要能完整跑通，就是非常关键的跨越。

成绩管理 待办清单 命令行记账本 调试能力
如何理解这 30 天： 这不是死记 30 个知识点，而是每天解决一个更真实的小问题。前面先学“怎么保存信息、怎么输出、怎么判断”，后面才能自然过渡到小项目。

6. 第 1 天正式课程：认识 Python 和程序

如果你是第一次学编程，第 1 天的目标不要定得太大。今天只需要理解“程序是什么”“代码是怎么执行的”，并且会用 `print()` 输出内容就够了。

1A

程序到底是什么

程序本质上就是一组写给电脑的明确步骤。你写得越清楚，电脑执行得越准确。

拿杯子 放咖啡粉 倒热水 搅拌
1B

代码默认如何执行

默认是从上到下、一句一句执行。这就叫顺序结构。

print("第一句") print("第二句") print("第三句")
1C

认识 `print()`

`print()` 的作用就是把内容输出到屏幕上。你可以把它理解成“让程序开口说话”。

print("你好，Python") print("欢迎开始学习编程")
  • 先把 `print` 记成“输出”。
  • 括号现在先接受它的写法，不用急着深究。
  • 字符串可以先理解成：被引号包起来的一段文字。
今天学完的标准： 你能自己写出 3 行 `print()`，并理解代码是按顺序执行的，就算第 1 天真正过关。

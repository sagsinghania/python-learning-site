def input_score(): try: return int(input("请输入分数：")) except ValueError: print("请输入数字") return None

本页目录

今天重点是回头修代码质量，不是继续加新功能。

1. 今天目标 2. 核心概念 3. 示例代码 4. 关键提醒 5. 今日练习 6. 今日总结

1. 今天目标

今天的目标是学会回头检查和整理项目：哪些逻辑重复了、哪些输入不稳、哪些命名不清楚、哪些函数可以再拆。

2. 核心概念

零基础最容易忽视的一件事是：代码第一次写出来，通常不是最好版本。重构和复盘不是额外负担，而是能力真正上升的过程。

Repeat

重复逻辑应该抽出来

如果一段输入保护、统计逻辑、展示逻辑出现多次，就应该考虑做成函数。

Stable

项目要防输入和边界问题

空列表、错误输入、文件不存在等情况，都值得回头补强。

3. 示例代码

这个例子说明：把输入保护抽成函数后，主流程会更干净。

def input_amount(): try: return float(input("请输入金额：")) except ValueError: print("金额必须是数字") return None
示例在说明什么： 重构不一定改变功能，但会明显改善代码的稳定性和可读性。

4. 关键提醒

  • 先找重复，再找混乱，再找不稳。
  • 不要一边重构一边大改需求。
  • 每次只处理一类问题，效果更清楚。
  • 重构后的代码最好重新跑一遍，确认功能没坏。
今天最该抓住的点： 重构的目标不是“变高级”，而是让程序更容易继续扩展和维护。

5. 今日练习

P1

找出一个重复逻辑

从你的项目里挑出一段重复逻辑，把它抽成单独函数。

P2

补一个输入保护

为项目里最容易出错的输入环节补上 try / except。

P3

整理一个长函数

把项目里偏长的一段函数拆成两段更清楚的小函数。

6. 今日总结

  • 重构是让项目从“能跑”进入“更稳”的关键一步。
  • 重复代码、输入边界和命名混乱最值得优先处理。
  • 重构不一定加功能，但会明显提升质量。
  • 这是从练习者走向更成熟写法的重要动作。
今天过关标准： 你能从已有项目里完成至少一处函数抽取、一处输入保护或一处命名整理，并说明为什么这样更好，就说明今天真正完成了调优复盘。

30 天的最后一天，不该再加新知识，而应该把方向彻底看清楚。

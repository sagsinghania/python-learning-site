1. 今天目标

今天的目标是做出一个能真正保存结果的小练习，让你从“会几个语法点”进入“能做一个小工具”。

2. 核心概念

文件项目练习的意义，不是功能多复杂，而是第一次让程序产生“下一次打开还能继续用”的感觉。

Save

保存输入内容

把用户输入的文字写进文件，让数据留在程序外部。

Read

重新读取历史记录

下次运行程序时，能够把之前保存的内容再显示出来。

3. 示例代码

下面是一个最小的学习记录工具：先追加一条记录，再把全部记录读出来。

note = input("请输入今天的学习记录：") with open("study_notes.txt", "a", encoding="utf-8") as file: file.write(note + "\n") with open("study_notes.txt", "r", encoding="utf-8") as file: print(file.read())
示例在说明什么： 程序不只是收一个输入然后结束，而是把内容保留下来，并能立刻看到历史记录。

4. 关键提醒

  • a 模式表示追加写入，适合记录型工具。
  • 项目的重点不是炫技，而是把输入、保存、显示这条链路跑通。
  • 保存后最好立刻读一次，确认结果确实落盘。
  • 这类项目特别适合反复改功能。
今天最该抓住的点： 真正的进步往往来自“把语法拼成小工具”，而不是继续无止境地看零散例子。

5. 今日练习

P1

学习记录本

让用户输入一条学习记录，保存到 txt 文件并显示全部历史记录。

P2

待办事项追加

把今天要做的 3 件事逐条追加到一个待办文件里。

P3

加入异常保护

如果文件读取出问题，给出友好提示，不要直接报错结束。

6. 今日总结

  • 文件项目练习是把基础语法拼起来的重要一步。
  • 追加写入比覆盖写入更适合日志和记录类工具。
  • 程序开始真正保存数据时，结果感会明显提升。
  • 小项目不必复杂，但链路要完整。
今天过关标准： 你能自己做出一个“输入内容 -> 写入文件 -> 读取显示”的最小工具，并确保它能重复使用，就说明今天真正过关。

项目一跑起来，下一步就该培养调试能力，不然问题一多就会卡住。

Day 12 / Loop Practice

循环真正有价值的时刻，
是它开始批量处理一组真实数据。

今天不再只是单独学一个语法点，而是把循环、列表和判断放到一起。你会第一次明显感受到，程序在“遍历数据并给出结果”。

你今天会看到的组合

一个循环里不只输出，还会顺手做判断和统计。

scores = [88, 59, 91, 73, 45] pass_count = 0 for score in scores: if score >= 60: pass_count = pass_count + 1 print("及格人数：", pass_count)

1. 今天目标

你今天的关键任务是学会“用一套逻辑处理一组数据”。这比单独写 `for`、单独写 `if` 更像真实程序，也更接近后面的小项目写法。

2. 成绩统计案例

先看一个最典型的组合案例：统计一组分数里有多少人及格。

scores = [88, 59, 91, 73, 45] pass_count = 0 for score in scores: if score >= 60: pass_count = pass_count + 1 print("及格人数：", pass_count)
为什么这个案例很关键： 它不再是“输出几次文字”，而是在遍历数据、检查条件、累计结果。这已经很接近真正的数据处理程序了。

3. 分步拆解这段程序

1

准备一组数据

`scores` 列表里放了多个分数，这是程序要处理的对象。

2

准备一个计数器

`pass_count = 0` 表示一开始还没有统计到任何及格人数。

3

逐个检查每个分数

循环每次拿到一个 `score`，再用 `if` 判断它是否及格。

4

条件成立就加 1

每发现一个及格分数，就把计数器加 1。循环结束后，结果自然就统计出来了。

4. 计数思路

很多入门程序都会用到“计数器”这个概念。它的套路很固定：先从 0 开始，满足条件就加 1。

先设成 0 循环检查 满足条件加 1 循环结束得到结果
你可以把它理解成记账： 每发现一个符合条件的对象，就在账本上加一次。最终这个数字就是统计结果。

5. 还能怎么升级

如果只统计及格人数已经会了，你还可以继续升级这段程序。

Up1

同时统计总分

再加一个 `total` 变量，每轮把当前分数累加进去，最后就能算平均分。

Up2

把不及格分数输出出来

在 `else` 分支里输出没及格的分数，程序就能告诉你哪些值需要重点关注。

6. 今日练习

P1

统计偶数个数

给一个数字列表，用循环和判断统计里面有多少个偶数。

P2

输出所有高分

准备一组分数，把大于等于 90 的分数单独输出出来。

P3

统计单词长度

给一个字符串列表，循环输出每个单词以及它的长度。

7. 今日总结

  • 循环真正常见的用法，是批量处理一组数据。
  • 列表提供数据集合，循环负责逐个处理。
  • 判断可以放在循环里筛选符合条件的项。
  • 计数器是从 0 开始、满足条件就加 1 的固定套路。
  • 这类组合题比单独语法更接近真实程序。
今天过关标准： 你能自己写出一个“遍历列表 + 条件判断 + 计数统计”的小程序，就说明今天真正把循环练熟了。

当你开始反复写相似逻辑时，下一步就该学会把它们封装起来。

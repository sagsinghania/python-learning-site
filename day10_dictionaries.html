1. 今天目标

今天的目标是学会把一条完整信息组织起来。前一天的列表更像“同类数据排成一排”，而字典更像“给每个数据贴上标签”。

2. 字典是什么

字典是由多个“键: 值”组成的结构。你可以把键理解成字段名，把值理解成字段内容。

例

学生资料

student = { "name": "小林", "age": 18, "city": "上海" }
想

它适合什么场景

联系人资料、商品信息、文章信息、账号设置，凡是“字段名明确”的资料，都很适合先放进字典。

先记外形： 字典通常用花括号 `{}`，里面是一组又一组的 `键: 值`。

3. 如何读取和修改

字典不是按位置取值，而是按键名取值。你想拿姓名，就直接写键名 `name`。

student = {"name": "小林", "age": 18} print(student["name"]) student["age"] = 19 print(student)
  • `student["name"]` 取出姓名。
  • `student["age"] = 19` 修改年龄。
  • 键名写错，程序就找不到对应值。

4. 如何增加新字段

字典可以边用边补内容。如果这条资料一开始没有城市，后面也能继续加进去。

student = {"name": "小林", "age": 18} student["city"] = "上海" print(student)
这一步很像现实资料整理： 先建立一条信息，再逐步把缺少的字段补全。

5. 列表和字典的区别

List

适合同类数据

比如 5 个名字、7 天温度、3 个价格。重点是“排成一排”。

Dict

适合一条完整资料

比如一个学生的姓名、年龄、分数。重点是“字段和内容对应起来”。

列表按位置找 字典按字段找 两者经常一起出现

7. 今日总结

  • 字典是键值对结构。
  • 字典适合表示一条有字段的完整资料。
  • 读取时用键名，不用索引位置。
  • 可以修改旧字段，也可以补充新字段。
  • 列表和字典经常一起用，但用途不同。
今天过关标准： 你能自己创建一个字典、取出其中一个字段、修改一个字段，再新增一个字段，就说明今天真正学会了字典的基础。

1. 今天目标

今天的目标是做出一个最小记账本：至少能保存消费项目和金额，并能显示记录和统计总支出。

2. 核心概念

当你能把“成绩管理器”的做法迁移到“记账本”，就说明你学到的不是一个案例，而是一套方法。

Record

每条账单是一条记录

项目名称和金额通常就是一条最小记录的核心字段。

Total

统计是最常见的项目需求

把所有金额加起来，立刻就能看出程序在真实解决问题。

3. 示例代码

下面这个骨架和昨天的成绩项目其实思路很像。

records = [] def add_record(records, item, amount): records.append({"item": item, "amount": amount}) def show_total(records): total = 0 for record in records: total += record["amount"] print("总支出：", total)
示例在说明什么： 你会发现，场景变了，但数据结构设计、函数拆分和统计方式仍然是同一套思路。

4. 关键提醒

  • 先确保金额是数字，再做统计。
  • 列表 + 字典仍然是很稳的基础结构。
  • 项目迁移能力比单个案例更重要。
  • 可以考虑把记账记录保存到文件，继续增强实用感。
今天最该抓住的点： 真正有价值的学习，不是会做一个例子，而是换场景后仍然知道该怎么拆。

5. 今日练习

P1

最小记账本

至少支持添加消费记录和显示总支出。

P2

加上类别字段

为记录再补一个 category 字段，比如餐饮、交通、学习。

P3

加上文件保存

把记账记录写入文件，让工具更像真实小项目。

6. 今日总结

  • 项目场景变了，但核心方法没变。
  • 列表、字典、函数和统计逻辑仍然是主力。
  • 迁移能力比死记案例更重要。
  • 这一天会帮助你摆脱“只能照着模板做”的状态。
今天过关标准： 你能做出一个最小记账本，并让它至少支持添加记录和统计总支出，就说明你的项目方法已经开始可迁移。
